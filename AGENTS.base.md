# AGENTS.base.md｜AI 协作开发通用检查清单

本文件是跨项目可复用的通用检查清单。具体项目应复制本文件，并在项目根目录 `AGENTS.md` 中追加项目定制部分。执行时以当前项目根目录 `AGENTS.md` 为准。

## A0. 执行入口

- [ ] 已读取当前项目根目录 `AGENTS.md`。
- [ ] 无法读取时，已暂停并要求用户提供文件内容。
- [ ] 已确认任务类型：需求设计 / 代码修改 / UI 调整 / PR 验收 / 文档治理 / 构建验证 / 其他。
- [ ] 已确认是否允许直接修改仓库；未确认时只输出方案。
- [ ] 涉及代码、配置、版本、数据库、构建或 PR 的变更前，已先输出设计方案。
- [ ] 用户明确确认后，才开始写入仓库。

## A1. 外部入口配置

- [ ] ChatGPT Project Instructions / Project Settings：已确认 / 未确认 / 不适用。
- [ ] 新会话启动词 / introductions：已确认 / 未确认 / 不适用。
- [ ] 项目入口文档 / project source：已确认 / 未确认 / 不适用。
- [ ] Codex / 自动化代理入口：已确认 / 未确认 / 不适用。
- [ ] PR 模板或人工 Code Review 清单：已确认 / 未确认 / 不适用。
- [ ] 无法确认外部配置时，已标注“未确认”。

## A2. 需求处理

- [ ] 已提炼需求目标。
- [ ] 已区分必须做、可选做、明确不做。
- [ ] 已识别架构、依赖、数据库、版本、大文件或破坏性风险。
- [ ] 已遵循最小必要修改原则。
- [ ] 需求不清时，已做最小合理假设；无法安全假设时已暂停确认。

## A3. 必读文件

- [ ] 已读取 README 或根目录说明。
- [ ] 已读取 handoff / design / requirements / changelog。
- [ ] 已读取 PR 模板、CI 配置和构建配置。
- [ ] 已读取本次任务直接相关源码、配置、脚本或文档。
- [ ] 涉及 GitHub connector 操作时，已读取项目内 connector guide。
- [ ] 文件缺失时，已标注“未发现 / 未确认”。

## A4. 方向一致性

- [ ] 修改符合项目产品定位。
- [ ] 修改符合当前架构主线。
- [ ] 修改符合用户已明确偏好的开发流程。
- [ ] 未引入与既有业务口径冲突的新规则。
- [ ] 未把临时修复伪装成长期架构。
- [ ] 未新增多个互相竞争的治理入口。

## A5. Git / 版本

- [ ] 已确认仓库地址和 source-of-truth 分支。
- [ ] 已确认默认分支、开发分支、任务分支和 PR 目标分支。
- [ ] 已确认是否需要创建任务分支。
- [ ] 已确认是否允许直接修改目标分支。
- [ ] 已确认是否涉及版本号。
- [ ] 涉及版本时，已同步项目要求的全部版本文件。
- [ ] 不改依赖时，未修改 lock 文件。
- [ ] 未执行强推，除非用户明确要求且已说明风险。

## A6. PR

- [ ] 已检查是否存在 PR 模板。
- [ ] PR body 包含执行前检查、修改目标、修改内容、验证状态、版本号、做了什么、没做什么、风险。
- [ ] 未在用户未要求时自动合并 PR。
- [ ] PR 目标分支符合项目规则。

## A7. 构建验证

- [ ] 已读取项目构建命令。
- [ ] 已按修改类型选择必要验证命令。
- [ ] 已执行可运行的验证命令。
- [ ] 无法执行时，已说明原因和替代检查。
- [ ] 未虚构 build / test / CI 通过。

## A8. 禁止事项

- [ ] 未做无关重构。
- [ ] 未顺手格式化大文件。
- [ ] 未删除用途未确认的代码、文档、表结构或配置。
- [ ] 未提交凭据、个人路径、日志、缓存、运行数据、数据库导出、安装包或构建产物。
- [ ] 未在无用户确认时修改依赖、lock 文件、CI、数据库结构或发布配置。
- [ ] 未把无法确认的信息写成“已确认”。

## A9. 交付

- [ ] 已说明修改目标、文件清单、核心逻辑、使用方式。
- [ ] 已说明验证方法、验证结果、未验证项和风险边界。
- [ ] 已说明是否修改版本号、依赖和 lock 文件。
- [ ] 已提供分支、commit、PR 或下载链接等交付入口。
- [ ] 如涉及 GitHub connector 操作，已检查是否出现新问题或更优流程；如有，已更新项目内 guide 或补充记录文件。
